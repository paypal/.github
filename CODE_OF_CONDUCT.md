# PayPal GitHub Public Repositories Code of Conduct

## Our Commitment

PayPal is committed to fostering an open, welcoming, and inclusive environment for all contributors to our public repositories. We endeavor to make participation in our projects a harassment-free experience for everyone, regardless of age, body size, disability, ethnicity, gender identity and expression, level of experience, nationality, personal appearance, race, religion, sexual identity and orientation, or other protected category.

## Our Standards

### Expected Behavior

Contributors to PayPal's public repositories are expected to:

- **Be respectful and inclusive**: Use welcoming and inclusive language in all interactions
- **Be collaborative**: Work constructively with other contributors and maintainers
- **Be professional**: Maintain a professional tone in all communications
- **Focus on the project**: Keep discussions relevant to the project's goals and technical merit
- **Accept constructive feedback**: Be open to feedback and different viewpoints
- **Show empathy**: Be understanding of different perspectives and experiences
- **Respect intellectual property**: Follow all applicable licenses and attribution requirements
- **Follow security practices**: Report security vulnerabilities responsibly through proper channels

Contributors should read and adhere to the GitHub Acceptable Use Policies.

## Scope

This Code of Conduct applies to:

- All PayPal public repositories on GitHub
- Issues, pull requests, and discussions within these repositories
- Comments on commits and code reviews
- Project documentation and wikis
- Community forums and chat channels associated with PayPal projects
- Public events and conferences when representing PayPal projects

## Reporting Guidelines

### How to Report

If you experience or witness unacceptable behavior, please report it by:

1. **For immediate safety concerns**: Contact GitHub Support directly
2. **For Code of Conduct violations**: Email: github-code-of-conduct@paypal.com
3. **For security vulnerabilities**: Follow our responsible disclosure process at paypal.com/security

### What to Include

When reporting, please provide:

- Your contact information
- Names of individuals involved
- Description of the incident
- Links to relevant content (issues, PRs, comments)
- Any additional context that would be helpful

### Confidentiality

All reports will be handled confidentially. We will not share reporter information without explicit consent, except as required by law or to ensure safety.

## Enforcement

### Investigation Process

1. **Acknowledgment**: Reports will be acknowledged within 48 hours
2. **Investigation**: PayPal will review the incident and gather additional information
3. **Decision**: Appropriate action will be determined based on the severity and context, in the sole discretion of PayPal
4. **Communication**: Involved parties will be notified of the outcome

### Possible Actions

Depending on the severity of the violation, actions may include:

- **Warning**: Private or public warning about inappropriate behavior
- **Temporary restriction**: Temporary limitation from commenting or contributing
- **Permanent ban**: Permanent removal from PayPal's GitHub community
- **Content removal**: Deletion of inappropriate content
- **Legal action**: In cases involving threatened or actual illegal activity

### Appeals Process

If you believe an enforcement action was taken in error, you may appeal by:

1. Emailing github-code-of-conduct@paypal.com within 30 days of notification of the enforcement action
2. Providing a clear explanation of why you believe the action was inappropriate
3. Including any relevant evidence or context

Appeals will be reviewed by maintainers not involved in the original decision.

## PayPal Responsibilities

PayPal is responsible for:

- **Modeling behavior**: Demonstrating the standards outlined in this Code of Conduct
- **Enforcement**: Taking appropriate action in response to violations
- **Communication**: Clearly communicating expectations and decisions
- **Fairness**: Applying this Code of Conduct consistently and fairly
- **Documentation**: Maintaining records of incidents and actions taken

## Community Guidelines

### Contributing

- Read and follow the project's contributing guidelines
- Test your changes thoroughly before submitting
- Write clear, descriptive commit messages and pull request descriptions
- Be patient with the review process and responsive to feedback
- Help newcomers get started and feel welcome

### Communication

- Use clear, professional language in all communications
- Stay on topic and keep discussions constructive
- Respect different time zones and response times
- Use appropriate channels for different types of discussions
- Search existing issues before creating new ones

### Recognition

We value all contributions to our projects, including:

- Code contributions and bug fixes
- Documentation improvements
- Testing and quality assurance
- Community support and mentoring
- Translations and accessibility improvements
